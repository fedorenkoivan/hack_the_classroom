# 🚀 SkillGap AI: Vector-Based Career Path Optimizer

**SkillGap AI** — це інтелектуальна аналітична платформа для розробників, яка використовує векторний пошук та Multi-Step Reasoning для точного виявлення прогалин між поточними навичками кандидата та реальними вимогами ринку праці.

---

## 📌 Яку проблему ми вирішуємо?

Традиційний пошук роботи та професійний розвиток у сфері IT страждають від трьох основних проблем:

1.  **Семантичний розрив:** Рекрутери та системи фільтрації (ATS) шукають конкретні ключові слова. Якщо розробник знає *FastAPI*, а у вакансії вказано *Flask*, система може помилково вважати їх несумісними.
2.  **Відсутність фокусу:** Навчальні плани університетів часто відірвані від динамічних трендів. Майбутні розробники витрачають час на нерелевантні технології, не розуміючи пріоритетів ринку.
3.  **Непрозорість прогресу:** Важко оцінити "вартість" навички — на скільки вивчення конкретної технології реально підвищить твій ринковий потенціал та дохід (Market Value).

## 📊 Аналіз ринку та конкурентів

| Рішення | Переваги | Чого бракує (The Gap) |
| :--- | :--- | :--- |
| **LinkedIn Skills** | Велика база вакансій, підбір за тегами. | **Відсутність семантики.** Не розуміє спорідненості технологій. Працює як жорсткий текстовий фільтр. |
| **LMS (Udemy/Coursera)** | Величезний вибір навчального контенту. | **Ангажованість.** Мета — продати курс, а не надати незалежну оцінку вашої готовності до вакансії. |
| **ATS-системи** | Ефективна фільтрація для HR-відділів. | **Candidate-blind.** Створені для відсіву кандидатів, а не для їхнього супроводу та професійного зростання. |

**SkillGap AI** заповнює цей розрив, стаючи на бік розробника (Candidate-first) та використовуючи векторну математику для об'єктивної оцінки досвіду.

---

## ⚙️ Технічний аналіз рішення (Як це працює)

Наш проєкт базується на архітектурі **Multi-Step AI Reasoning** та векторному просторі навичок.

### 1. Побудова графа навичок (Vector Space)
Ми використовуємо модель `text-embedding-3-small` від OpenAI для перетворення кожної навички у вектор. Завдяки розширенню **pgvector** у PostgreSQL, ми розраховуємо косинусну схожість між ними.
* *Логіка:* Система розуміє, що вектор "NestJS" знаходиться близько до "Express.js", тому автоматично зараховує частину досвіду як релевантну.

### 2. Market-Driven Ingestion
Ми агрегуємо дані в реальному часі через **Adzuna API** та **Jooble API**. Це дозволяє системі базуватися не на теоретичних вимогах, а на "живому" попиті роботодавців (наприклад, актуальні вакансії компанії Poster).

### 3. Алгоритм аналізу (Multi-Step Pipeline)
1.  **Extraction:** LLM (GPT-4o-mini) витягує сутності (Hard Skills, Soft Skills, Experience) з тексту вакансії та CV користувача.
2.  **Comparison:** Векторний двигун проводить `Semantic Matching`, знаходячи реальні прогалини, які не перекриваються схожими навичками.
3.  **Monetization:** Система зіставляє знайдені прогалини з медіанними зарплатами ринку, розраховуючи **"The Cost of a Skill"**.

### 4. Project-Based Roadmap
Замість посилань на теорію, AI генерує унікальне технічне завдання для міні-проєкту. Користувач заповнює прогалину через практику, одночасно створюючи артефакт для портфоліо.

## 🛠 Технологічний стек

* **Frontend:** React 19, Tailwind CSS, **React Flow** (інтерактивне Skill Tree), **Recharts** (Radar Charts для порівняння).
* **Backend:** Node.js (Express), Prisma ORM.
* **Database:** PostgreSQL + **pgvector** (Supabase).
* **AI/ML:** OpenAI SDK (Embeddings + Chat Completion).
* **Data Sources:** Adzuna API, Jooble API.

## 🌟 Авторська фішка: "Market Anomaly Detector"
Ми впровадили алгоритм виявлення аномалій ринку. Система аналізує динаміку появи нових технологій у вакансіях. Якщо попит на *Zustand* або *Docker* зростає на 30%+ за тиждень, користувач отримує сповіщення: *"Це новий тренд. Опануй це зараз, щоб випередити конкурентів"*.
