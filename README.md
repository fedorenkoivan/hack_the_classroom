# 🎓 Hack the Classroom — SkillPath

**SkillPath** — це веб-додаток для розробників-початківців, який допомагає об'єктивно оцінити свій рівень знань JavaScript і TypeScript, та отримати персональний roadmap із реальних GitHub-репозиторіїв для подальшого зростання.

| | URL |

## 💡 Яку проблему ми вирішуємо?

Починаючий розробник часто не розуміє:
- **Де він зараз** — Junior чи Middle?
- **Що вчити далі** — з тисяч ресурсів складно обрати правильні
- **Як виміряти прогрес** — суб'єктивна самооцінка дає хибні результати

**SkillPath** вирішує це через короткий квіз по кожному обраному скілу, автоматичне визначення рівня та готовий список GitHub-репозиторіїв, підібраних саме під цей рівень.

---

## ✨ Чому це рішення якісне?

- **Без реєстрації** — анонімна сесія через `localStorage`, нічого зайвого
- **Персоналізовано** — окремий результат і roadmap для кожного скіла
- **Реальні дані** — репозиторії зберігаються в БД, легко оновлювати без коду
- **Збереження прогресу** — результати записуються в PostgreSQL, можна повернутись
- **Масштабовано** — архітектура дозволяє легко додавати нові скіли та запитання
- **Продакшн-ready** — деплой на Railway (API + PostgreSQL) та Vercel (Frontend)

### `GET /api/repos?skillSlug=js&level=junior`
Повертає репозиторії для конкретного скіла та рівня.

**Query параметри:**
| Параметр | Обов'язковий | Значення |
|---|---|---|
| `skillSlug` | ✅ | `js` або `ts` |
| `level` | ✅ | `junior` або `middle` |

**Відповідь:**
```json
[
  {
    "id": "...",
    "name": "javascript-algorithms",
    "url": "https://github.com/trekhleb/javascript-algorithms",
    "level": "junior",
    "skillId": "..."
  }
]
```

---

### `POST /api/results`
Зберігає результати квізу та повертає персональний roadmap із репозиторіями.

**Тіло запиту:**
```json
{
  "sessionId": "sess_1234567890_abc",
  "results": [
    { "skillSlug": "js", "score": 42, "level": "Junior" },
    { "skillSlug": "ts", "score": 78, "level": "Middle" }
  ]
}
```

**Відповідь:**
```json
{
  "userId": "clx...",
  "skills": [
    {
      "skillSlug": "js",
      "label": "JavaScript",
      "score": 42,
      "level": "Junior",
      "repos": [
        { "name": "javascript-algorithms", "url": "https://github.com/..." }
      ]
    }
  ]
}
```

**Логіка рівнів:**
| Відсоток | Рівень | Репозиторіїв |
|---|---|---|
| < 45% | Junior | 20 |
| 45–74% | Middle | 30 |
| ≥ 75% | Strong Middle | 30 |

## 🖥️ Функціонал фронтенду

### Крок 1 — Вибір скілів
- Картки з доступними скілами (JavaScript, TypeScript)
- Мультиселект — можна обрати обидва одночасно
- Кнопка "Почати квіз" активна тільки якщо обрано хоча б один скіл

### Крок 2 — Квіз
- По **3 запитання** на кожен обраний скіл (всього до 6 запитань)
- Кожне запитання має **5 варіантів відповіді** зі score від 1 до 5
- Лічильник прогресу: `відповіді / загальна кількість`
- Відповіді зберігаються в локальному стані, можна змінити до завершення

### Крок 3 — Результати та Roadmap
- **Загальний score** у відсотках
- **Рівень** — окремо для кожного скіла
- **Roadmap** — список GitHub-репозиторіїв з прямими посиланнями (20 або 30 залежно від рівня)
- **Автоматичне збереження** в БД після завершення квізу
- Індикатор стану збереження: `Зберігаємо…` → `✓ Збережено в БД`

### Технічні деталі фронтенду
- **Анонімна сесія** — унікальний `sessionId` генерується і зберігається в `localStorage`
- **Без перезавантаження** — весь стан управляється через React hooks
- **Захист від подвійного збереження** — `useRef` гарантує один POST-запит на квіз
- **Responsive** — адаптивна верстка для мобільних і десктопу

## 📌 Яку проблему ми вирішуємо?

Традиційний пошук роботи та професійний розвиток у сфері IT страждають від трьох основних проблем:

1.  **Семантичний розрив:** Рекрутери та системи фільтрації (ATS) шукають конкретні ключові слова. Якщо розробник знає *FastAPI*, а у вакансії вказано *Flask*, система може помилково вважати їх несумісними.
2.  **Відсутність фокусу:** Навчальні плани університетів часто відірвані від динамічних трендів. Майбутні розробники витрачають час на нерелевантні технології, не розуміючи пріоритетів ринку.
3.  **Непрозорість прогресу:** Важко оцінити "вартість" навички — на скільки вивчення конкретної технології реально підвищить твій ринковий потенціал та дохід (Market Value).

**SkillGap AI** заповнює цей розрив, стаючи на бік розробника (Candidate-first) та використовуючи векторну математику для об'єктивної оцінки досвіду.

## ⚙️ Технічний аналіз рішення (Як це працює)

Наш проєкт базується на архітектурі **Multi-Step AI Reasoning** та векторному просторі навичок.

### 1. Побудова графа навичок (Vector Space)
Ми використовуємо модель `text-embedding-3-small` від OpenAI для перетворення кожної навички у вектор. Завдяки розширенню **pgvector** у PostgreSQL, ми розраховуємо косинусну схожість між ними.
* *Логіка:* Система розуміє, що вектор "NestJS" знаходиться близько до "Express.js", тому автоматично зараховує частину досвіду як релевантну.

### 2. Market-Driven Ingestion
Ми агрегуємо дані в реальному часі через **Adzuna API** та **Jooble API**. Це дозволяє системі базуватися не на теоретичних вимогах, а на "живому" попиті роботодавців (наприклад, актуальні вакансії компанії Poster).

### 3. Алгоритм аналізу (Multi-Step Pipeline)
1.  **Extraction:** LLM (GPT-4o-mini) витягує сутності (Hard Skills, Soft Skills, Experience) з тексту вакансії та CV користувача.
2.  **Comparison:** Векторний двигун проводить `Semantic Matching`, знаходячи реальні прогалини, які не перекриваються схожими навичками.
3.  **Monetization:** Система зіставляє знайдені прогалини з медіанними зарплатами ринку, розраховуючи **"The Cost of a Skill"**.

### 4. Project-Based Roadmap
Замість посилань на теорію, AI генерує унікальне технічне завдання для міні-проєкту. Користувач заповнює прогалину через практику, одночасно створюючи артефакт для портфоліо.
